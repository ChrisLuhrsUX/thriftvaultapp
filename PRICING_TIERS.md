# ThriftVault Pricing Tier Registry

Single source of truth for the weekly drift-watch routine (`trig_0113xK23HSeSpB46ySDYJtBF`, Mon 9am ET). Edit via PR. The routine fetches this file every Monday, computes the week's 10-tier slice using the rotation rule below, queries each platform for recent sold comps, and opens a PR with a drift report at `pricing-drift-reports/YYYY-MM-DD.md`.

The routine does NOT modify `services/gemini.ts`. Band edits are reviewed and applied manually by the user.

## Rotation rule

```
WEEK_INDEX = ISO week number of run date (US Eastern)

Pools sorted by id ascending. Slice with wrap-around using (pool + pool):
  HOT_SLICE  = HOT_POOL[(WEEK_INDEX * 6) mod len(HOT)..][:6]
  WARM_SLICE = WARM_POOL[(WEEK_INDEX * 3) mod len(WARM)..][:3]
  COOL_SLICE = COOL_POOL[(WEEK_INDEX * 1) mod len(COOL)..][:1]

Report on HOT_SLICE + WARM_SLICE + COOL_SLICE = 10 tiers.
```

The doubling-and-slicing trick (`(pool + pool)[start:start+count]`) handles wrap without modular index math. The algorithm is a pure function of (registry, ISO week), so it is stateless and deterministic.

## Pools

| Pool | Size | Stride | Cycle |
| --- | --- | --- | --- |
| HOT  | 12 | 6 | every 2 weeks |
| WARM | 24 | 3 | every 8 weeks |
| COOL | 49 | 1 | every 49 weeks |

Pool sizes are flexible. The algorithm degrades gracefully when counts shift. Promote / demote freely in future PRs.

## Schema

| field | meaning |
| --- | --- |
| `id` | 3-digit zero-padded, stable forever. The rotation hashes on `id`, so renumbering breaks history. Insertions append; deletions retire the id. |
| `tier_name` | Human-readable. Matches `services/gemini.ts` phrasing. |
| `category` | `handmade` \| `factory_apparel` \| `furniture` \| `jewelry` \| `code_clamp` |
| `current_band` | Exact band from `services/gemini.ts`. No literal `\|` in cells (use parens). |
| `depop_query` / `poshmark_query` / `ebay_query` | Quoted search strings. For furniture rows, substitute Chairish / 1stDibs / Etsy where Depop or Poshmark don't apply, prefix the cell with `(Chairish)` etc. For `code_clamp` rows, all three cells = `n/a` and the routine switches to narrative sanity-check mode. |
| `volatility` | `HOT` \| `WARM` \| `COOL`. Drives rotation. |
| `notes` | Short context (active spike, recently calibrated, etc.). |
